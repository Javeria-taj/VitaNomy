The Types File — Your Sync Bible

Write this together in Hour 1. If either person needs to change a type, message first. This is what keeps you from breaking each other's code.

🔒 Rule

TypeScript enforces the contract automatically. If backend changes a response shape without telling you, the compiler will error on your component immediately. This is your early-warning system — don't silence TypeScript errors, fix them.

// /types/patient.ts — SHARED. Both teammates import from here.

export interface PatientInput {
  name: string
  age: number
  gender: 'male' | 'female' | 'other'
  weight: number        // kg
  height: number        // cm
  systolic_bp: number   // e.g. 120
  diastolic_bp: number  // e.g. 80
  glucose: number       // mg/dL fasting
  cholesterol: number   // mg/dL total
  smoking: boolean
  exercise: 'none' | 'light' | 'moderate' | 'heavy'
  family_history: boolean
}

export interface RiskScores {
  diabetes: number      // 0–100
  cardiac: number       // 0–100
  hypertension: number  // 0–100
}

export interface AnalyzeResponse {
  patient_id: string
  risk_scores: RiskScores
  overall_risk: 'LOW' | 'MEDIUM' | 'HIGH' | 'CRITICAL'
  insights: string[]
  recommendations: string[]
}

export interface SimulateRequest {
  patient: PatientInput
  scenario: 'exercise' | 'diet' | 'quit_smoking' | 'medication'
}

export interface SimulateResponse {
  original_risks: RiskScores
  projected_risks: RiskScores
  delta: RiskScores        // projected - original (negative = improvement)
  narrative: string
}

export interface ChatMessage {
  role: 'user' | 'assistant'
  content: string
}

export interface ChatRequest {
  patient: PatientInput
  analysis: AnalyzeResponse
  history: ChatMessage[]
  message: string
}

export interface ChatResponse {
  reply: string
}

Zustand Store

// /store/patientStore.ts
import { create } from 'zustand'
import { PatientInput, AnalyzeResponse, SimulateResponse, ChatMessage } from '@/types/patient'

interface PatientStore {
  patient: PatientInput | null
  analysis: AnalyzeResponse | null
  simulation: SimulateResponse | null
  chatHistory: ChatMessage[]
  isLoading: boolean
  setPatient: (p: PatientInput) => void
  setAnalysis: (a: AnalyzeResponse) => void
  setSimulation: (s: SimulateResponse) => void
  addChatMessage: (m: ChatMessage) => void
  setLoading: (v: boolean) => void
  reset: () => void
}

export const usePatientStore = create<PatientStore>((set) => ({
  patient: null, analysis: null, simulation: null,
  chatHistory: [], isLoading: false,
  setPatient: (patient) => set({ patient }),
  setAnalysis: (analysis) => set({ analysis }),
  setSimulation: (simulation) => set({ simulation }),
  addChatMessage: (m) => set((s) => ({ chatHistory: [...s.chatHistory, m] })),
  setLoading: (isLoading) => set({ isLoading }),
  reset: () => set({ patient: null, analysis: null, simulation: null, chatHistory: [] })
}))

Risk Engine — Pure Logic, No AI

Deterministic scoring means instant responses. Claude is only used for narrative text, not numbers. Judges see fast results.

// /lib/riskEngine.ts
import { PatientInput, RiskScores } from '@/types/patient'

function calcBMI(weight: number, height: number) {
  return weight / ((height / 100) ** 2)
}

export function calculateRisks(p: PatientInput): RiskScores {
  const bmi = calcBMI(p.weight, p.height)

  // — Diabetes
  let diabetes = 0
  if (p.glucose >= 126) diabetes += 40
  else if (p.glucose >= 100) diabetes += 20
  if (bmi >= 30) diabetes += 20; else if (bmi >= 25) diabetes += 10
  if (p.family_history) diabetes += 20
  if (p.exercise === 'none') diabetes += 15
  if (p.smoking) diabetes += 5

  // — Cardiac
  let cardiac = 0
  if (p.cholesterol >= 240) cardiac += 30
  else if (p.cholesterol >= 200) cardiac += 15
  if (p.systolic_bp >= 140) cardiac += 25
  if (p.smoking) cardiac += 25
  if (p.exercise === 'none') cardiac += 15
  if (p.age >= 45) cardiac += 10
  if (p.family_history) cardiac += 15

  // — Hypertension
  let hypertension = 0
  if (p.systolic_bp >= 140) hypertension += 40
  else if (p.systolic_bp >= 130) hypertension += 20
  if (p.diastolic_bp >= 90) hypertension += 20
  if (bmi >= 30) hypertension += 15
  if (p.smoking) hypertension += 10
  if (p.age >= 50) hypertension += 10
  if (p.exercise === 'none') hypertension += 5

  return {
    diabetes: Math.min(diabetes, 95),
    cardiac: Math.min(cardiac, 95),
    hypertension: Math.min(hypertension, 95)
  }
}

export function getOverallRisk(s: RiskScores) {
  const max = Math.max(s.diabetes, s.cardiac, s.hypertension)
  if (max >= 75) return 'CRITICAL'
  if (max >= 55) return 'HIGH'
  if (max >= 35) return 'MEDIUM'
  return 'LOW'
}

Claude Service

// /lib/claudeService.ts
import Anthropic from '@anthropic-ai/sdk'
import { PatientInput, RiskScores, ChatMessage } from '@/types/patient'

const client = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY })

export async function getInsights(patient: PatientInput, risks: RiskScores) {
  const msg = await client.messages.create({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 600,
    messages: [{
      role: 'user',
      content: `You are a clinical AI. Patient: ${patient.age}yr ${patient.gender}
Risk scores — Diabetes: ${risks.diabetes}, Cardiac: ${risks.cardiac}, Hypertension: ${risks.hypertension}
Vitals: BP ${patient.systolic_bp}/${patient.diastolic_bp}, Glucose ${patient.glucose}, Cholesterol ${patient.cholesterol}
Lifestyle: Smoking=${patient.smoking}, Exercise=${patient.exercise}
Return ONLY valid JSON (no markdown):
{"insights": ["...","...","..."], "recommendations": ["...","...","..."]}`
    }]
  })
  return JSON.parse((msg.content[0] as any).text)
}

export async function chatWithTwin(
  patient: PatientInput, context: string,
  history: ChatMessage[], message: string
) {
  const msg = await client.messages.create({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 400,
    system: `Compassionate AI health assistant for ${patient.name}'s digital twin.
Context: ${context}. Be concise. Never diagnose. Never alarm.`,
    messages: [...history, { role: 'user', content: message }] as any
  })
  return (msg.content[0] as any).text
}

Analyze Route

// /app/api/analyze/route.ts
import { NextRequest, NextResponse } from 'next/server'
import { PatientInput, AnalyzeResponse } from '@/types/patient'
import { calculateRisks, getOverallRisk } from '@/lib/riskEngine'
import { getInsights } from '@/lib/claudeService'

export async function POST(req: NextRequest) {
  try {
    const patient: PatientInput = await req.json()
    const risk_scores = calculateRisks(patient)
    const overall_risk = getOverallRisk(risk_scores)
    const { insights, recommendations } = await getInsights(patient, risk_scores)

    const response: AnalyzeResponse = {
      patient_id: `${patient.name}-${Date.now()}`,
      risk_scores, overall_risk, insights, recommendations
    }
    return NextResponse.json(response)
  } catch (err) {
    return NextResponse.json({ error: 'Analysis failed' }, { status: 500 })
  }
}

Contingency
Risk Mitigation

Every hackathon has surprises. Here's how to handle the most common ones without losing hours.

Risk Severity Mitigation
Claude API slow or rate-limited HIGH Cache last 3 responses in memory. Have 5 pre-generated insights as fallback strings. Risk scores still work instantly — only text is affected.
Backend not ready at Hour 12 MED Frontend works fully on mock data indefinitely. Delay integration to Hour 15. Demo can still run on mocks if needed.
Type shape mismatch on integration MED TypeScript surfaces this immediately. Fix in the types file, let compiler guide both sides. Budget 30 min for this.
Feature creep past Hour 15 HIGH Hard freeze at Hour 22. Write the freeze time on a sticky note. New ideas go in a "v2" list — don't touch the code.
Typing live data during pitch HIGH Pre-load Alex Morgan data and save as a URL param or localStorage. Never enter form data live on stage.
npm server crashes during demo MED Keep a terminal window open at all times. Deploy to Vercel at Hour 20 as a live backup URL.
API key exposed in code HIGH Always use process.env.ANTHROPIC_API_KEY. Confirm .env.local is in .gitignore before first push. Check with git status.
